# ハンディ端末モジュール（Pi Zero 2 W）

Pi Zero 2 W 上で動作する送信ソフトウェア、`mirrorctl` 設定、キュー管理スクリプトなどを再構築する領域。OnSiteLogistics リポジトリのコードを参考にしつつ、RaspberryPiServer との連携を明確に分離する。

## 再構築の観点
- API 送信ロジックをモジュール化し、Pi5 のエンドポイント変更へ柔軟に対応できるようにする。
- `mirrorctl` / `mirror_compare` の設定テンプレートを整備し、14 日検証のための状態管理を簡素化する。
- 手動テストハンドブックのシナリオに沿ってログ取得・障害対応手順を記録する。
